# Support

## Public repositories

Use the repository's issue tracker for reproducible documentation problems and defects
in public CodeVelo tooling. Search existing issues before opening a new one and include
the version, environment, expected behavior, actual behavior, and reproduction steps.

## CodeVelo services and project inquiries

GitHub issues are not a customer-support or project-intake channel.

- Website: [codevelo.dev](https://codevelo.dev)
- Contact: [hello@codevelo.dev](mailto:hello@codevelo.dev)
- Discovery call: [codevelo.dev/book](https://codevelo.dev/book)

## Security

Report suspected vulnerabilities privately using [SECURITY.md](SECURITY.md). Do not
open a public issue.
