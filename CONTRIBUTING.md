# Contributing to CodeVelo public repositories

Thank you for helping improve CodeVelo's public documentation and tooling.

## Before opening a contribution

1. Check the repository README and open issues for repository-specific guidance.
2. Use an issue to discuss substantial changes before investing in an implementation.
3. Keep each pull request focused on one clear outcome.
4. Add or update tests and documentation when behavior changes.
5. Do not include credentials, private infrastructure details, client information, or
   material you do not have the right to contribute.

## Pull requests

A good pull request includes:

- a concise explanation of the problem and solution;
- verification steps or test results;
- documentation for user-visible behavior;
- disclosure of breaking changes or operational considerations; and
- links to relevant issues without exposing private project context.

Submission of a pull request does not guarantee acceptance. CodeVelo may decline work
that does not align with the repository's scope, security requirements, or maintenance
capacity.

## Conduct

Participation is governed by our [Code of Conduct](CODE_OF_CONDUCT.md).

## Security reports

Do not open a public issue for a suspected vulnerability. Follow the private reporting
instructions in [SECURITY.md](SECURITY.md).
